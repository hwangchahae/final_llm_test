🎯 요약
------------------------------------------------------------
  - 회의에서는 XML 형식을 사용한 데이터베이스 설계에 대해 논의했으며, 타임라인, 스피커 ID, 시간 정보 등을 포함하는 구조를 논의했다. 전화 수준과 프레임 수준의 데이터 인코딩 방식에 대해 이진 방식을 고려하는 의견이 나왔으며, 파일 형식과 데이터베이스 선택에 대한 고민이 있었다.
  - 기존 도구(ATLAS, NIST)와의 호환성 및 향후 확장성을 고려한 설계가 필요하다는 의견이 나왔으며, 데이터 공개 시 주석 처리 가능성과 유연한 표현 방식의 중요성이 강조되었다.
  - 시간 구간과 태그의 관계, 주석 겹침, 형식 확장성, 도구 개발 가능성 등에 대해 논의했으며, XML의 구조화된 검색 기능과 텍스트/격자 동형성에 대한 논의가 있었다.

📌 안건
------------------------------------------------------------

  📍 1. **XML 형식의 데이터베이스 구조 설계**: 타임라인, 스피커 ID, 시간 정보 등을 포함한 구조를 결정하고, 태그 기반의 유연한 정보 추가 방식을 논의한다.

  📍 2. **전화 수준과 프레임 수준의 데이터 인코딩 방식 결정**: 이진 방식과 텍스트 기반 방식의 장단점을 비교하고, 파일 시스템 제한(예: 2GB)을 고려한 최적의 인코딩 방식을 선택한다.

  📍 3. **기존 도구(ATLAS, NIST)와의 호환성 검토**: 기존 형식과의 호환성, 향후 확장성, 번역기 활용 가능성 등을 검토하여 표준화된 형식을 결정한다.
